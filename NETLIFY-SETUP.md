# Запуск сайта на Netlify — пошагово «для чайников»

Эта инструкция для **того, кто настраивает сайт один раз** (разработчик или помощник Елены).  
**Елене** эту инструкцию читать не нужно — ей достаточно файла **ELENA-ADMIN.md**.

---

## Что получится в итоге

- Сайт открывается по адресу `https://ваш-домен.ru`
- Заявки с форм приходят на email
- Елена правит контент по адресу `https://ваш-домен.ru/admin/`
- Домен можно купить на **reg.ru**, а сайт будет работать через Netlify (это нормально)

---

## Что понадобится

1. Компьютер с интернетом  
2. Папка **`site`** с файлами сайта (она у вас на диске)  
3. Бесплатная регистрация на **GitHub.com** (хранилище кода)  
4. Бесплатная регистрация на **Netlify.com** (хостинг)  
5. Домен на reg.ru (можно подключить позже)

Время: **1–2 часа** в первый раз.

# ЧАСТЬ 1. GitHub — положить сайт в «облако»

Netlify и админка работают только если код лежит в GitHub.

## Шаг 1.1. Создать аккаунт GitHub

1. Откройте https://github.com/signup  
2. Зарегистрируйтесь (email, пароль).  
3. Подтвердите email из письма.

## Шаг 1.2. Создать новый репозиторий

1. В GitHub нажмите **«+»** → **New repository**.  
2. **Repository name:** `elena-bocharova-site` (любое латинское имя).  
3. **Public** — оставьте.  
4. **НЕ** ставьте галочки README, .gitignore, license.  
5. **Create repository**.

## Шаг 1.3. Загрузить папку site на GitHub

### Вариант А — через программу GitHub Desktop (проще всего)

1. Скачайте **GitHub Desktop**: https://desktop.github.com/  
2. Установите, войдите в свой аккаунт GitHub.  
3. **File → Add local repository** → укажите папку `site`.  
   - Если пишет «не git-репозиторий» → нажмите **create a repository**.  
4. Внизу слева в поле **Summary** напишите: `Первый запуск сайта`.  
5. Нажмите **Commit to main**.  
6. Нажмите **Publish repository** (опубликовать).  
7. Снимите галочку **Keep this code private**, если хотите бесплатный простой вариант, или оставьте private — Netlify поддерживает оба.

### Вариант Б — через командную строку (если умеете)

```bash
cd "путь/к/папке/site"
git init
git add .
git commit -m "Первый запуск сайта"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/elena-bocharova-site.git
git push -u origin main
```

**Проверка:** на сайте github.com в репозитории видны папки `assets`, `content`, `index.html`.

# ЧАСТЬ 2. Netlify — опубликовать сайт

## Шаг 2.1. Регистрация

1. https://app.netlify.com/signup  
2. Выберите **Sign up with GitHub** — так проще связать сайт с кодом.  
3. Разрешите доступ Netlify к GitHub.

## Шаг 2.2. Подключить репозиторий

1. **Add new site** → **Import an existing project**.  
2. **Deploy with GitHub**.  
3. Выберите репозиторий `elena-bocharova-site`.  
4. Настройки сборки (должны подтянуться из `netlify.toml`):

   | Поле | Значение |
   |------|----------|
   | Branch to deploy | `main` |
   | Build command | `python scripts/netlify_build.py` |
   | Publish directory | `.` |

5. **Deploy site**.

6. Подождите 2–5 минут. Статус станет **Published**.

7. Netlify даст временный адрес вида `random-name-123.netlify.app` — откройте его в браузере. Должна открыться главная страница.

**Если сборка упала (Failed):**  
- Site configuration → Build & deploy → проверьте build command.  
- Deploy log — ищите красную ошибку; чаще всего нет Python — в `netlify.toml` уже указан Python 3.11.

# ЧАСТЬ 3. Формы — чтобы заявки приходили на почту

1. В Netlify: ваш сайт → **Site configuration** → **Forms**.  
2. Должны появиться формы: `excursion`, `coaching`, `strategy` и др. (после первого успешного деплоя).  
3. **Form notifications** → **Add notification** → **Email notification**.  
4. Email: `Elenabbocharova@yandex.ru` (или нужный).  
5. Сохраните.

**Проверка:** откройте сайт → раздел экскурсий → отправьте тестовую заявку. Письмо должно прийти за 1–2 минуты.

# ЧАСТЬ 4. Админка для Елены (Netlify Identity + Git Gateway)

> **Где вы сейчас:** слева **Project configuration** (зелёная точка) → в центре блок **Identity** с пунктами Overview / Users / Registration / Emails / Services.  
> Старые названия вроде «Settings and usage» в Netlify **больше нет** — ниже актуальный интерфейс.

## Шаг 4.1. Identity (вход по email)

### A. Включить Identity (если ещё не включено)

1. Слева: **Project configuration**
2. В центральной колонке найдите блок **Identity**
3. Если видите кнопку **Enable Identity** — нажмите её  
   *(У вас Identity уже включён, если открывается страница «Identity settings» с API endpoint.)*

### B. Только по приглашению (Invite only)

1. Оставаясь в **Project configuration**
2. В блоке **Identity** нажмите **Registration** (не Overview!)
3. Найдите **Registration preferences** или **Allow registration**
4. Выберите **Invite only** / **Only invited users**
5. **Save** / **Update settings**

### C. Пригласить Елену

1. В блоке **Identity** нажмите **Users**  
   *(или на Overview прокрутите до секции Users — кнопка **Invite users**)*
2. **Invite users**
3. Email Елены → **Send invite** / **Invite**
4. **Важно:** в письме **Accept invite** → должна открыться **`/admin/`** с формой **задать пароль**.  
   Если перекинуло на главную и пароль не спросили:
   - **Сразу:** в письме **ПКМ по ссылке** → **Копировать адрес** → вставьте в браузер, но путь замените на  
     `https://ВАШ-САЙТ.netlify.app/admin/` (хвост `#invite_token=...` оставьте)
   - **Или:** залейте обновлённый `site.js` на GitHub → снова **Invite users** (главная сама перекинет в `/admin/`)
5. Задайте пароль → дальше только `/admin/` → **Log in**

## Шаг 4.2. Git Gateway (чтобы админка сохраняла изменения)

1. **Project configuration**
2. Блок **Identity** → **Services**
3. **Git Gateway** → **Enable Git Gateway**
4. Если попросит — подтвердите доступ к GitHub

## Шаг 4.3. Проверка админки

1. Откройте `https://benevolent-sherbet-0bada3.netlify.app/admin/` (ваш netlify-адрес + `/admin/`)
2. **Log in** → email, на который приглашали
3. Слева: **Экскурсии**, **Подкаст** и т.д.
4. Мелкая правка → **Опубликовать**
5. Через 2–3 мин **Deploys** → Published → обновите сайт (F5)

Подробнее для Елены — **ELENA-ADMIN.md**.

# ЧАСТЬ 5. Свой домен с reg.ru

Домен покупаете на reg.ru. Сайт продолжает жить на Netlify — вы только «прикрепляете» имя.

## Шаг 5.1. В Netlify

1. **Domain management** → **Add a domain** → введите `kodmesta.ru` (ваш домен).  
2. Netlify покажет, какие DNS-записи нужны.

## Шаг 5.2. В reg.ru

1. Личный кабинет → ваш домен → **DNS-серверы / Ресурсные записи**.  
2. Обычно Netlify просит:
   - Запись **A** на IP Netlify (`75.2.60.5`), **или**
   - **CNAME** для `www` на `ваш-сайт.netlify.app`
3. Netlify часто рекомендует использовать **NS-серверы Netlify** — тогда в reg.ru меняете NS на те, что дал Netlify (проще для новичка).

4. Подождите **от 15 минут до 24 часов** — DNS обновляется не мгновенно.

5. В Netlify включится **HTTPS** (замочек) автоматически.

## Шаг 5.3. Прописать домен в сайте

1. Откройте `content/site.json` в GitHub (или через админку → Контакты → SEO).  
2. Поле **siteUrl:** `"https://kodmesta.ru"` (ваш домен, с `https://`).  
3. Сохраните / закоммитьте — Netlify пересоберёт сайт.

Или локально запустите `python scripts/apply_seo.py` и залейте изменения на GitHub.

# ЧАСТЬ 6. Яндекс.Метрика (по желанию)

1. Создайте счётчик на https://metrika.yandex.ru  
2. Скопируйте **номер счётчика**.

Счётчик заработает только у посетителей, нажавших «Принять все» в баннере cookie.

# Частые проблемы

| Проблема | Решение |
|----------|---------|
| Админка открывается, но не сохраняет | Включите **Git Gateway** (часть 4.2) |
| Елена не может войти | **Identity → Invite users** — отправьте приглашение снова |
| Форма не отправляется | Проверьте **Forms** в Netlify; сайт должен быть задеплоен, не открыт как файл с диска |
| Сайт старый после правок в админке | Подождите 3 мин, нажмите F5; проверьте **Deploys** — последний деплой должен быть Success |
| Картинки не грузятся | Загружайте через кнопку **«Выбрать»** в админке, не вставляйте путь руками |
