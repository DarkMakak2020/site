# Сайт Елены Бочаровой — мультибрендовый хаб «Тихая роскошь»

Полноценный многостраничный сайт личного бренда: главная-хаб + разделы направлений,
контент-коллекции (экскурсии, подкаст, книги, журнал), формы заявок, SEO и слой админки.

Сайт **статический** (HTML + CSS + JS, без сборки). Это сознательный выбор: на машине нет
Node.js/npm/git, а статика работает сразу, идеальна для SEO и сохраняет премиальный дизайн
макета 1‑в‑1. Реальная админка подключается через Decap CMS при деплое (см. ниже).

---

## 1. Как запустить локально

Сборка не нужна. Нужен любой статический сервер. Проще всего — Python (уже установлен):

```bash
cd "site"
python -m http.server 8123
```

Откройте в браузере: <http://127.0.0.1:8123/>

> Важно: открывать именно через сервер (`http://...`), а не двойным кликом по файлу
> (`file://`). Сайт использует корне-относительные ссылки (`/kod-mesta/`) и загрузку
> JSON через `fetch` — это работает только по http.

## 3. Как редактировать контент (без программиста)

Весь контент лежит в папке `content/` в виде JSON. Есть два способа.

### Способ A. Визуальная админка (рекомендуется для Елены)
После деплоя на Netlify (раздел 6) откройте `https://ваш-домен/admin/` — это панель
**Decap CMS**: разделы «Экскурсии», «Подкаст», «Книги», «Журнал», «Настройки сайта».
Заполняете поля, нажимаете «Опубликовать» — изменения сохраняются в репозиторий и сайт
обновляется автоматически.

### Способ B. Прямое редактирование JSON (для разработчика)
Открыть нужный файл в `content/`, добавить объект в массив `items`. Пример новой экскурсии
(`content/excursions.json`):

```json
{
  "slug": "novyy-marshrut",
  "title": "Название маршрута",
  "summary": "Короткое описание для карточки",
  "fullDescription": "Полное описание",
  "duration": "2 часа",
  "format": "Пешая · малая группа",
  "audience": "Частные гости",
  "price": "от 4 000 ₽",
  "cover": "/assets/img/uploads/photo.jpg",
  "tags": ["Центр"],
  "published": true
}
```

- `published: false` — скрыть материал, не удаляя его.
- `cover` пустой → карточка покажет красивую монограмму-заглушку.
- Журнал: поле `slug` формирует ссылку `/journal/article.html?slug=...`.

Глобальные данные (email, соцсети, SEO, ID счётчиков) — в `content/site.json`.

## 4. Формы и заявки

Все формы используют единый слой `assets/js/forms.js` (функция `sendToProvider`).
Сейчас провайдер — **Netlify Forms**: заявки приходят на e-mail и в дашборд Netlify,
ничего настраивать не нужно после деплоя. Локально форма покажет дружелюбный фолбэк
с предложением написать на почту (т.к. backend'а нет).

Поля форм: имя, телефон, email, направление интереса, комментарий, согласие на обработку ПДн.

Формы на сайте: диагностическая/стратегическая сессия и HR-консалтинг (`/strateg/`),
коучинг (`/coach/`), предзапись на практикум (`/praktikum-hr/`), экскурсия и корпоративная
экскурсия (`/kod-mesta/excursions/`), гость подкаста (`/kod-mesta/podcast/`),
общий контакт (`/about/`).

### Подключить CRM / другой приёмник (без переписывания сайта)
В `assets/js/forms.js` поменяйте `CONFIG.provider`:
- `"netlify"` — Netlify Forms (по умолчанию);
- `"webhook"` — POST JSON на свой `webhookUrl` (свой backend, Zapier/Make, AmoCRM, Битрикс24, Google Sheets);
- `"mailto"` — открыть почтовый клиент (резервный вариант).

Разметка форм и UX при этом не меняются.

## 5. SEO

- У каждой страницы свои `title`, `description`, canonical, Open Graph.
- Schema.org: `Person` (главная), `Course` (практикум).
- `sitemap.xml` и `robots.txt` готовы — **замените `example.com` на реальный домен**
  (также в тегах canonical/og:url и в `content/site.json → seo.siteUrl`).
- Человекопонятные URL (`/kod-mesta/excursions/`).
- OG-картинка по умолчанию: `assets/img/og-default.svg`.

> Ограничение статики: карточки коллекций (экскурсии/подкаст/книги/журнал) рендерятся
> на клиенте из JSON. Их списки и страница статьи индексируются хуже, чем статический
> HTML. Если нужен максимальный SEO по этим материалам — см. раздел 8 (апгрейд на Astro).

## 6. Деплой и админка (Netlify + Decap CMS)

1. Залейте папку `site/` в git-репозиторий (GitHub/GitLab).
2. На [netlify.com](https://www.netlify.com) → «Add new site» → подключите репозиторий.
   Build command — пусто, publish directory — корень сайта (`netlify.toml` уже настроен).
3. Включите **Netlify Identity**: Site settings → Identity → Enable.
   - Registration: Invite only (чтобы доступ был только у Елены).
   - Services → **Git Gateway** → Enable.
4. Пригласите Елену по e-mail (Identity → Invite users).
5. Готово: `https://ваш-домен/admin/` — вход по приглашению, визуальное редактирование.

Формы Netlify включатся автоматически (в разметке есть `data-netlify="true"` и скрытые поля).

## 7. Аналитика и реклама

В `content/site.json → analytics` есть поля `yandexMetrikaId` и `vkPixelId`.
- Заполните `yandexMetrikaId` — счётчик Яндекс.Метрики подключится автоматически
  (код в `assets/js/content.js → injectAnalytics`).
- VK Pixel — заложено место (TODO: вставить официальный сниппет при получении ID).
- Цели на отправку форм отправляются автоматически (`form_<имя>`), плюс событие в `dataLayer`.
- UTM-метки приходят в URL и попадают в данные заявки (`_page`).

## 9. Дизайн-система (кратко)

Палитра «Тихая роскошь» (в `assets/css/base.css`): глубокий изумруд, бордо/вино, тёплый
айвори, графит, золото — только микродетали. Шрифты: Cormorant Garamond (заголовки) +
Switzer (текст). Запрещены жёлтый/оранжевый/яркое золото, stock-look, Tilda-look. При
правках держитесь токенов — это сохраняет премиальность.
